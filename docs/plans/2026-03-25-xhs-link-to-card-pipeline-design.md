# XHS Link to Card Pipeline 推广卡片设计（Cyber Dark）

日期：2026-03-25

## 目标
做一套 6 页 Cyber Dark 信息流卡片，用于推广 XHS-link-to-card-pipeline Skill，完整传达价值、流程、规范与安装方式。

## 受众与场景
- 需要把链接/文档转成可发布素材包的小红书运营与内容创作者
- 想要稳定产出知识卡片的工具型用户

## 信息架构与页序（6 页）
1. 封面：Skill 名称 + 一句话价值主张 + 3 个卖点
2. 输出清单：摘要 / SVG 卡片 / PNG 发布图 / 文案
3. 标准流程：5 步流程线
4. 分页结构模板：封面→机制→步骤→清单→总结
5. 满铺无留白规范：3 条硬规则
6. 安装方式：一句话 + GitHub 链接（“将这个链接发给 agent 安装即可”）

## 视觉与版式
- 风格：Cyber Dark（深色底 + 霓虹渐变强调 + 轻微 glow）
- 版式：标题区固定高度；内容区 2 列网格；重点用霓虹描边框
- 统一标识：右上角小字显示 `XHS-link-to-card-pipeline`

## 逐页文案
1. 封面
- 主标题：XHS Link → Card Pipeline
- 副标题：把链接/文档变成可发素材包
- 三卖点：摘要 / 卡片 / 文案

2. 输出清单
- 结构化摘要：一句话结论 + 关键点
- SVG 卡片：封面 + 内页（4–6 张）
- PNG 发布图：1242×1660 无留白
- 小红书文案：标题/标签/简介/正文骨架/CTA

3. 标准流程
- 解析 → 规划 → 选风格 → 画 SVG → 导出 PNG

4. 分页结构模板
- 封面价值主张 → 机制/方法 → 步骤拆解 → 清单/路线 → 总结/CTA

5. 满铺无留白规范
- width=1242 height=1660 + viewBox
- 第一层必须有满铺 rect 背景
- 圆角/阴影在安全区内

6. 安装方式
- 标题：安装方式
- 一句话：将这个链接发给 agent 安装即可
- 链接：https://github.com/leefrane-creator/XHS-link-to-card-pipeline
- 小字：GitHub 安装 | 一句话搞定

## 实施约束
- 画布严格 1242×1660，首层必须有满铺 rect
- 生成 SVG 后再批量导出 PNG

## 交付物
- 6 张 SVG（Cyber Dark）
- 6 张 PNG（1242×1660）
- 小红书文案包（标题/标签/简介/正文骨架/CTA）
