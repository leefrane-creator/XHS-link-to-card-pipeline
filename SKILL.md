# XHS Link → Card Pipeline（可发小红书资产包）

## 你会得到（输出清单）
- 结构化摘要：一句话结论 + 3–5 个关键点 + 适用人群/场景
- SVG 卡片：封面 + 内页（建议 4–6 张）
- PNG 发布图：从 SVG 批量导出，尺寸固定 `1242×1660`（3:4）
- 小红书文案：标题（多版本）、标签、简介、正文骨架、CTA

## 标准流程
1) 解析链接/文档，提炼卖点与结构（面向“小红书读者”的表达）
2) 规划分页：封面价值主张 → 关键机制/方法 → 步骤拆解 → 清单/路线 → 总结/CTA
3) **先选风格（必选）** → 之后所有卡片保持同一套视觉风格
4) 生成 SVG 卡片（严格遵守“满铺无留白”规范）
5) 运行脚本导出 PNG：`scripts/convert_svg_to_png.ps1`
6) 汇总为可发布素材包（PNG + 文案）

## 2.5) 先选风格（必选，4 选 1）
开始制图前，必须先问用户选择下面 4 套预设之一（选定后全套卡片保持一致）：
1. Cyber Dark（暗黑霓虹 / AgentOps）：深色底 + 霓虹渐变强调 + 轻微 glow
2. Editorial Paper（杂志纸感 / 研究笔记）：米白纸张质感 + 红色标注 + 轻颗粒
3. Swiss Minimal（瑞士极简 / 信息图）：黑白灰网格 + 单一强调色（建议红）+ 强对齐
4. Bold Gradient Poster（大胆渐变海报）：大面积渐变 + 超大标题 + 3 个卖点

交互话术示例：
> 这套图你想走哪种风格？1 暗黑霓虹 / 2 杂志纸感 / 3 瑞士极简 / 4 大胆渐变海报

## SVG 规范（必须，解决“空白/黑边/没铺满”）
PNG 出现边角空白/黑边，99% 是因为 **SVG 画布/背景没有满铺** 或导出时使用了 `contain` 导致 letterbox。

每张 SVG 必须满足：
- 画布固定：`width="1242" height="1660"`，并使用 `viewBox="0 0 1242 1660"`
- **背景满铺**：第一层绘制 `rect x="0" y="0" width="1242" height="1660"`（颜色/渐变随风格变化）
- 不要在最外层留任何 `margin/padding`；圆角/阴影必须在安全区内呈现，不得“挤出画布”
- 图片/截图（如产品界面）一律用 **cover** 思路裁切（中心优先），避免左右/上下留白

## 导出 PNG（强制 1242×1660，避免留白/黑边）
在包含 `.svg` 的目录运行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/convert_svg_to_png.ps1
```

- 默认会递归寻找 `*.svg` 并输出到 `png_output/`
- 该脚本使用 Edge Headless 渲染并强制“铺满”，目标是 **完全没有边缘留白或黑边**
- 若仍出现空白：优先检查 SVG 是否缺失“背景满铺 rect”或画布尺寸不对，而不是在 PNG 后期补边
