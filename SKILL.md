# XHS Link → Card Pipeline（可发小红书资产包）

## 你会得到（输出清单）
- 结构化摘要：一句话结论 + 3–5 个关键点 + 适用人群/场景
- SVG 卡片：封面 + 内页（建议 4–6 张）
- PNG 发布图：从 SVG 批量导出，尺寸固定 `1242×1660`（3:4）
- 小红书文案：标题（多版本）、标签、简介、正文骨架、CTA

## 标准流程
1) 解析链接/文档，提炼卖点与结构（面向“小红书读者”的表达）
2) 规划分页：封面价值主张 → 关键机制/方法 → 步骤拆解 → 清单/路线 → 总结/CTA
3) **先选风格（必选）** → 之后所有卡片保持同一套视觉风格
4) 生成 SVG 卡片（严格遵守“满铺无留白”规范）
5) 运行脚本导出 PNG：`scripts/convert_svg_to_png.ps1`
6) 汇总为可发布素材包（PNG + 文案）

## 2.5) 先选风格（必选，8 选 1）
开始制图前，必须先问用户选择下面 8 套预设之一（选定后全套卡片保持一致）：
1. Cyber Dark（暗黑霓虹 / AgentOps）：深色底 + 霓虹渐变强调 + 轻微 glow
2. Editorial Paper（杂志纸感 / 研究笔记）：米白纸张质感 + 红色标注 + 轻颗粒
3. Swiss Minimal（瑞士极简 / 信息图）：黑白灰网格 + 单一强调色（建议红）+ 强对齐
4. Bold Gradient Poster（大胆渐变海报）：大面积渐变 + 超大标题 + 3 个卖点
5. Brutalist Mono（野性单色）：黑白高对比 + 粗线条 + 巨型编号
6. Sticker Pop（贴纸潮流）：高饱和撞色 + 贴纸标注 + 手写感标签
7. Data Terminal（终端数据感）：等宽字体 + 栅格分区 + 仪表/表格卡块
8. Neo Newspaper（新报纸）：报纸分栏 + 标题层级强 + 局部强调底纹

交互话术示例：
> 这套图你想走哪种风格？1 暗黑霓虹 / 2 杂志纸感 / 3 瑞士极简 / 4 大胆渐变海报 / 5 野性单色 / 6 贴纸潮流 / 7 终端数据感 / 8 新报纸

## 风格 DNA（必须，不允许只换颜色）
每套风格必须同时定义并锁定以下维度，禁止“同一版式只换配色”：
- 布局结构：中轴海报 / 左对齐信息流 / 分栏报纸 / 栅格仪表
- 字体系统：标题字体、数字字体、正文风格（无衬线 / 等宽 / 报纸衬线）
- 形状语言：圆角卡片 / 直角线框 / 斜切块 / 贴纸气泡
- 背景纹理：纯色 / 颗粒 / 网格 / 噪点 / 纸感
- 信息密度：大标题低密度 or 中密度条目 or 高密度数据卡
- 装饰规则：每种风格允许与禁止的元素（如 glow、描边、批注、贴纸）

## 模板池规则（必须）
- 每套风格至少维护 3 套版式模板：封面、榜单/拆解、总结/CTA
- 模板在同一风格内可轮换，但不得跨风格复用同一骨架
- 允许同主题内容复用文案，不允许复用“完全同构版式”

## 风格验收清单（必须）
- 风格内一致：同一套图的字体、形状、间距、装饰一致
- 风格间可分辨：任意两套风格在缩略图层面也应明显不同
- 结构差异达标：至少 2 个维度（布局/字体/形状/纹理）发生变化
- 失败处理：若风格相似度过高，优先改版式和字体，再调颜色

## SVG 规范（必须，解决“空白/黑边/没铺满”）
PNG 出现边角空白/黑边，99% 是因为 **SVG 画布/背景没有满铺** 或导出时使用了 `contain` 导致 letterbox。

每张 SVG 必须满足：
- 画布固定：`width="1242" height="1660"`，并使用 `viewBox="0 0 1242 1660"`
- **背景满铺**：第一层绘制 `rect x="0" y="0" width="1242" height="1660"`（颜色/渐变随风格变化）
- 不要在最外层留任何 `margin/padding`；圆角/阴影必须在安全区内呈现，不得“挤出画布”
- 图片/截图（如产品界面）一律用 **cover** 思路裁切（中心优先），避免左右/上下留白

## 边缘一致性硬约束（新增，必须）
- 最终成图四边（尤其右侧与底部）不得出现与主体背景不一致的白边/黑边断层
- 若导出器存在可视区偏差，必须通过“满铺补偿”或后处理裁切回填，保证视觉上与整体背景同色
- 背景基色需与当前风格一致（不要求固定某个单一色值），并确保导出后四角像素与该风格边缘背景一致
- 交付前必须做验收：检查 `1242×1660` 尺寸 + 四角像素一致 + 肉眼无边框感

## 导出 PNG（强制 1242×1660，避免留白/黑边）
在包含 `.svg` 的目录运行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/convert_svg_to_png.ps1
```

- 默认会递归寻找 `*.svg` 并输出到 `png_output/`
- 默认导出策略：**放大截图后固定裁切到 `1242×1660`**（几何法，跨任务更稳定）
- 不再依赖“颜色阈值补边”作为主链路，避免新风格/新版式触发误判
- 该脚本内置验收：尺寸检查、边缘连续性检查、右/下边带检测（必要时修复）
- 若仍出现空白：优先检查 SVG 是否缺失“背景满铺 rect”或画布尺寸不对，而不是在 PNG 后期补边
- 若仍出现边缘断层：必须修正导出方式或做裁切回填，直到边缘与整体背景一致

## 风格未选中的兜底策略
- 若用户未明确选择 8 套风格之一，默认使用 `Cyber Dark`，并在输出中显式注明“本次使用默认风格”
